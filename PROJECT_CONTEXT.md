# 월급실험실 프로젝트 컨텍스트

## 프로젝트 소개

월급실험실은 대한민국 현실 생활 조건을 시뮬레이션하는 웹 서비스이다.

사용자는 자신의 연봉, 주거 형태, 생활 스타일 등의 조건을 입력하여 아래와 같은 현실적인 생활 가능성을 분석할 수 있다.

* 서울 자취 가능 여부
* 퇴사 후 생존 가능 기간
* 월세 vs 전세 비교
* 자차 유지 가능 여부
* 청약 현실성 분석
* 저축 가능 수준 분석

본 프로젝트는 단순 계산기가 아니라 “현실적인 의사결정 보조 플랫폼”을 목표로 한다.

---

# 핵심 철학

## 1. 계산보다 해석이 중요하다

단순 숫자 계산이 아니라 다음과 같은 현실적인 해석을 제공해야 한다.

예시:

* 월세 비중이 실수령 대비 높습니다.
* 현재 소비 구조에서는 저축 여력이 낮습니다.
* 퇴사 전 비상금 확보가 필요합니다.

---

## 2. 로그인 없는 즉시 사용 경험

사용자는 자신의 연봉/자산 데이터를 저장하는 것을 부담스러워할 가능성이 높다.

따라서 초기 MVP는 다음 원칙을 따른다.

* 로그인 없이 즉시 사용 가능
* 민감 데이터 기본 저장 금지
* 결과 저장은 선택 기능
* 사용자 정보 최소 수집

---

## 3. 공공데이터 기반 현실 시뮬레이션

서비스는 대한민국 공공데이터를 기반으로 동작한다.

예상 데이터:

* 국토교통부 전월세 실거래가
* 생활물가 데이터
* 청약 관련 데이터
* 정부지원금 데이터

단순 조회가 아니라:

* 데이터 정제
* 지역별 평균 계산
* 조건 기반 분석
* 시뮬레이션 리포트

를 수행한다.

---

# UI 기획 방향

## 제품 성격
월급실험실은 단순 계산기 사이트가 아니라 사용자의 월급과 생활 조건을 기반으로 현실적인 삶의 선택지를 시뮬레이션하는 웹 플랫폼이다.

핵심은 숫자 계산보다 현실적인 해석과 리포트에 있다.

## 핵심 원칙
- 계산기 사이트처럼 보이면 안 된다.
- 긴 입력폼, 엑셀 느낌 UI, 정부 사이트 느낌, 복잡한 표 중심 UI를 피한다.
- 생활 리포트 느낌, 질문형 UX, 직관적인 카드 UI, 쉬운 해석 중심 결과를 우선한다.
- 메인 페이지는 상단 탭보다 카드형 시뮬레이션 선택 화면을 우선한다.
- 각 시뮬레이션은 독립 페이지로 구성한다.
- 입력 UX는 한 번에 모든 필드를 보여주지 않고 단계형 또는 질문형으로 구성한다.
- 결과 화면은 가능성 등급, 예상 생활비, 월세 비중, 저축 가능 금액, 주의 포인트, 추천 행동을 포함한 리포트처럼 보여준다.

## 디자인 톤앤매너
원하는 분위기:
- 토스 스타일의 쉬운 금융 UX
- 뱅크샐러드 스타일의 리포트 UI
- 오늘의집 스타일의 생활 친화적인 감성

원하지 않는 분위기:
- 관리자 페이지
- ERP
- 통계 대시보드
- 공공기관 사이트

## 우선 UI 작업
현재는 실제 백엔드 연동보다 UI/UX 흐름 설계가 우선이다.

우선 구현 목표:
1. 메인 카드형 시뮬레이션 선택 화면
2. 서울 자취 가능성 페이지
3. 단계형 입력 UX
4. 결과 리포트 Mock UI
5. 반응형 레이아웃
6. 기본 디자인 시스템 정리

아직 실제 계산 로직 구현은 하지 않는다.

# Frontend 아키텍처 원칙

## 기본 원칙

* Server Component 우선 사용
* 입력 폼만 Client Component 사용
* 계산 로직은 프론트에 두지 않는다
* 타입 안정성을 중요하게 유지한다

## MVP 기능

1. 서울 자취 가능성 시뮬레이션

등으로 확장한다.

# Plan 로드맵

초기 개발은 아래 Plan 순서를 따른다.

1. 프로젝트 기본 세팅
2. 프론트엔드 Mock 기반 시뮬레이션 선택 화면과 서울 자취 가능성 UX
3. 백엔드 서울 자취 가능성 시뮬레이션 API
4. Docker Compose 기반 로컬 통합 실행

초기 프론트엔드는 Mock 데이터로 UI 흐름을 먼저 완성하고, 이후 백엔드 API 계약에 맞춰 연동한다.

후속 시뮬레이션은 아래 Plan으로 분리한다.

* 퇴사 후 생존 가능 기간
* 월세 vs 전세 비교
* 자차 유지 가능성
* 청약 현실성
* 정부지원금 또는 보조금 추천
